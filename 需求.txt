# 网络小说平台开发需求文档
**版本**：1.0  
**最后更新**：2026-07-05

---

## 1. 项目概述

### 1.1 项目定位
一个聚焦**高质量类型文学**（非套路文）的去中心化网络小说平台。强调**作者自主定价**、**版权归属作者**、**社区共治**，并通过**文件系统存储**实现高性能、低成本的运维架构。

### 1.2 核心理念
- **作者友好**：作品属于作者，可随时导出、迁移。
- **读者归属**：按文学类型（硬科幻、推演文学等）构建深度阅读社区。
- **技术开放**：对个人爬虫友好，通过技术手段区分商业爬虫。
- **安全底线**：前端确认、后端限流、内容净化三位一体。

---

## 2. 技术架构

### 2.1 整体架构
采用 **前后端分离** 与 **动静分离** 架构：

| 层级 | 技术栈 | 职责 |
| :--- | :--- | :--- |
| **前端** | Vue 3 + Vite + Element Plus | 用户界面、交互逻辑、状态管理 |
| **后端** | Go (Gin/Fiber) | 核心 API 服务（认证、评论、支付、跨域检测） |
| **静态服务** | Nginx | 直接返回小说 `.html` 正文、`index.json`、图片等静态资源 |
| **数据库** | MySQL (或 PostgreSQL) | 存储用户、评论、打赏记录等动态数据 |
| **缓存** | Redis | 章节列表缓存、限流计数、会话状态 |
| **文件存储** | 操作系统文件系统 | 小说正文（`.html`）、章节索引（`index.json`）、作者配置、封面图 |

> **技术决策说明**：  
> - 小说正文以 HTML 文件存储于文件系统，数据库仅存路径索引。  
> - 此举将数据库读压力降低约 80%，且使“作品打包导出”成为天然功能。  
> - 核心后端使用 Go，文件转换等偶尔任务可后续视需求引入 Python 作为独立辅助进程。

### 2.2 目录结构
```
/data/novels/
├── authors/
│   ├── {author_id}/
│   │   ├── {novel_id}/
│   │   │   ├── index.json          # 章节索引 + 元数据
│   │   │   ├── 1.html              # 第1章正文
│   │   │   ├── 2.html              # 第2章正文
│   │   │   └── cover.jpg           # 封面图
│   │   └── config.json             # 作者个性化配置
│   └── ...
└── uploads/                         # 用户上传资源（头像、附件）
```

### 2.3 关键设计原则
- **配置外部化**：所有敏感配置（数据库密码、JWT 密钥）通过 `.env` + 系统环境变量注入，严禁硬编码。
- **无状态设计**：服务端不存储用户会话，使用 JWT 进行身份认证。
- **读写分离**：Nginx 处理所有静态资源读取，Go 后端仅处理动态请求。

## 3. 功能模块

### 3.1 用户系统

#### 3.1.1 注册与登录
- 支持邮箱/手机号注册。
- 密码使用 `bcrypt` 哈希存储。
- 登录成功后下发 JWT，存储于 `HttpOnly; Secure; SameSite=Strict` Cookie 中。
- **安全机制**：
  - 登录失败 5 次后，IP 或用户名锁定 15 分钟。
  - 强制 HTTPS，配置 HSTS。

#### 3.1.2 用户分级与权限

| 用户类型 | 权限说明 |
| :--- | :--- |
| **未登录访客** | 可浏览作品前 30%（试读），不可评论、不可打分 |
| **普通用户（已登录）** | 完整阅读、评论、评分、收藏、打赏 |
| **作者** | 发布作品、自主定价、导入/导出作品、自定义小说页面样式 |
| **VIP 作者** | 额外解锁自定义 CSS + 沙箱化 JS 动画权限（需审核） |
| **社区仲裁员** | 由社区选举产生，处理违规举报与争议 |

### 3.2 作品管理

#### 3.2.1 作品发布
- 作者通过 Web 编辑器创作，支持 **富文本** 与 **Markdown** 两种模式，一键切换。
- **Markdown 模式**支持：
  - LaTeX 公式（行内 `$...$`，块级 `$$...$$`）
  - 颜色标记（`[红色文本]{red}` 语法，配合取色器）
- 作品需选择 **主分类**，可选填 **标签**。

#### 3.2.2 分类体系
采用 **主分类 + 标签** 结构，主分类如下：
- 硬科幻、奇幻、推演文学、架空历史、现实主义、悬疑推理、实验文学、同人区、其他。

> 平台不设立“爽文”“水文”等分类。调性通过分类体系与推荐算法自然过滤。

#### 3.2.3 定价与收益
- **作者自主定价**，按每千字或每章设定价格。
- 平台抽成 **10%** 用于服务器运营，剩余归作者。
- 读者可通过 **打赏** 直接支持作者，打赏金额全额转交作者（扣除支付通道手续费）。

#### 3.2.4 导入与导出
- **导出**：作者可一键导出作品为 **ZIP**（含 HTML + JSON）、**Markdown**、**EPUB**、**TXT**。
- **导入**：支持上传 **ZIP**、**Markdown**、**TXT** 文件，系统自动解析章节结构并生成标准格式。
  - **鲁棒解析**：支持 `第X章`、`Chapter X`、分割线、YAML Front Matter 等多种格式。
  - 导入后提供 **预览确认界面**，作者可手动调整章节划分。

### 3.3 阅读与交互

#### 3.3.1 试读机制
- 未登录用户可免费阅读作品前 **30%** 或前 **5章**（取较小者）。
- 完整阅读需登录。

#### 3.3.2 评论系统
- 支持 **章评/段评**（选中文本评论）与 **书评**。
- **跨域评论警告**：从同人区跨域进入政治文学区的用户，首次评论前触发两级警告：
  1. 文化差异提醒
  2. 法律风险告知（需手动输入“我承诺承担法律责任”）
- **评论限速**：
  - 跨域访客在政治文学区：**1分钟/条**
  - 原生政治用户：**15-30秒/条**
  - 单日违规次数过多 → 递进式加重限制（5分钟 → 30分钟 → 只读模式）

#### 3.3.3 内容确认机制（心理缓冲）
进入 **同人区** 或 **政治文学区** 时触发确认弹窗：
- **首次进入**：3次确认（告知 → 警告 → 主动输入确认）。
- **跨域移动**（同人区 ↔ 政治文学区）：5次确认（含文化差异预警）。
- **技术实现**：确认层由 `<xxx.js>` 控制，HTML 内容**明文存在**。爬虫不触发确认，禁用 JS 或查看源代码即可绕过——**绕过即视为知情同意**。

#### 3.3.4 评价与排名
- **多维度评分**：类型完成度、叙事质量、思想深度、社区口碑、更新稳定性。
- **排名独立**：按分类展示“本周最佳”“本月新作”，不设全站总榜（仅设“年度社区之选”）。
- **新书保护期**：每部新作品上线后，在所属分类的“新书榜”展示 **30天**，确保获得初始曝光。

### 3.4 社区与论坛

#### 3.4.1 子论坛与大论坛双轨制
- **子论坛**（毛细血管）：每个作品、每个分类下的“流派”都有独立讨论区。
- **大论坛**（公共广场）：类型文学理论研讨、社区议事厅、跨界灵感碰撞。

#### 3.4.2 作者交流区
- 认证作者可进入半封闭交流区，包含：设定研讨室、逻辑检验台、创作急诊室、资源共享库。
- **作者互评**：同行评议加权计入作品评分。

#### 3.4.3 微论坛
- 嵌入式轻讨论组件，支持即兴话题、作者问答间、灵感速记板。

### 3.5 作者个性化美化

#### 3.5.1 页面定制权限

| 作者等级 | 定制范围 | 审核要求 |
| :--- | :--- | :--- |
| 普通作者 | 预设主题、封面图、基础配色 | 免审 |
| 认证作者 | 自定义 CSS（白名单属性） | 首次需审核 |
| VIP 作者 | 完整 CSS + 沙箱化 JS 动画 | 首次需审核，后续自动过审（连续3次无违规） |

#### 3.5.2 安全实现
- CSS 通过白名单净化器过滤（仅允许 `.novel-*` 选择器、安全属性，禁用 `url()`、`@import`）。
- JS 在 `<iframe sandbox="allow-scripts">` 中隔离执行，禁止访问父页面、Cookie、LocalStorage。
- 平台提供预置动画库供作者调用，避免直接执行任意 JS。

## 4. 安全策略

### 4.1 SQL 注入防御
- **强制使用参数化查询**（Prepared Statements），所有用户输入通过 `?` 占位符传递。
- 数据库账号采用最小权限原则。

### 4.2 XSS 防御
| 内容类型 | 处理方式 |
| :--- | :--- |
| 小说正文（作者） | 白名单净化（`bluemonday`），允许 `<p>`、`<strong>` 等安全标签 |
| 用户评论 | 全局 HTML 实体转义（`html.EscapeString`） |
| 用户信息（昵称等） | 全局 HTML 实体转义 |

### 4.3 图片上传安全
- **重新编码存储**：上传的图片经 `image.Decode` 解码后，用 `webp.Encode` 重新生成，丢弃所有元数据与隐藏块。
- 图片目录通过 Nginx 配置禁止执行任何脚本。

### 4.4 反爬虫策略
- **限流**：单 IP 每分钟请求超过 **100次** 触发限流（返回 429）。
- **区分爬虫类型**：
  - 个人爬虫（`python-requests`、`curl` 等）→ 放行，受限流约束。
  - 商业爬虫（高频、全站扫描）→ 封禁或下发验证码挑战。
- **矩阵爬虫**（多 IP、多账号）→ 触发惩罚性赔偿条款（按协议执行）。

### 4.5 数据追踪与版权保护
- **不可见水印**：所有页面内容嵌入零宽字符水印，与用户账号绑定。
- **蜜罐数据**：在页面中嵌入正常用户不会访问的隐藏链接，访问即确认为爬虫。

## 5. 治理与合规

### 5.1 社区公约
- 作者保留作品完整版权，平台仅获得运营所需授权。
- 用户行为准则：
  - 禁止发布违法违规内容。
  - 禁止跨区引战、恶意举报。
  - 禁止将爬取数据用于商业目的。

### 5.2 分级访问与实名制
| 区域 | 访问条件 |
| :--- | :--- |
| 普通小说区 | 无需登录（可试读） |
| 同人区 | 登录 + 3次确认 |
| 政治文学区 | 登录 + 实名认证 + 3次确认 |
| 跨域移动（同人→政治） | 额外 5次确认 |

### 5.3 违规处理流程
- **个人爬虫售卖数据**：赔偿作者经济损失。
- **商业爬虫**：赔偿作者 + 赔偿平台服务器成本。
- **商业伪装个人**：赔偿作者 + 赔偿平台服务器成本 + 惩罚性赔偿。
- **恶意引战/攻击**：封禁账号，保留法律追诉权利。

## 6. 性能与运维要求

### 6.1 性能指标
- 页面首屏加载时间 < 1.5s（含静态资源）。
- API 接口 P99 延迟 < 200ms。
- 单机支持并发连接数 > 5000。

### 6.2 部署要求
- 支持 Docker 容器化部署。
- 支持水平扩展（静态资源与后端分离，后端无状态）。
- 日志统一采集，支持 ELK 或 Loki 检索。

### 6.3 备份策略
- 数据库每日自动备份。
- 文件系统每 6 小时增量备份，每日全量备份。
- 备份文件保留 30 天。

## 9. 站长面板（Admin Panel）

### 9.1 概述
站长面板是平台运营管理的核心工具，面向平台管理员（站长）及社区运营人员。聚焦于**内容监管、用户管理、财务透明、系统监控**四大职能。

> 设计原则：所有操作留痕，所有数据可导出，所有敏感操作需二次确认。

### 9.2 面板结构

#### 9.2.1 仪表盘（Dashboard）
- **实时概览**：
  - 今日新增用户 / 作品 / 评论 / 打赏
  - 当前在线人数（最近15分钟活跃）
  - 服务器负载（CPU / 内存 / 磁盘 / 带宽）
  - 最近1小时异常请求数（触发限流的IP数量）
- **待处理事务**：
  - 待审核作品数（含样式/脚本审核）
  - 待处理举报数（按严重程度排序）
  - 待处理的VIP申请数

#### 9.2.2 内容管理
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **作品审核** | 浏览所有待审核作品列表，按分类/状态筛选 | 通过 / 退回（附原因）/ 下架（紧急） |
| **作品管理** | 全局搜索、筛选、排序所有已发布作品 | 查看详情、修改分类、强制下架、删除作品（需二次确认） |
| **分类管理** | 增/删/改主分类与子分类 | 调整分类顺序、合并分类、设置分类图标/颜色 |
| **标签管理** | 管理全站标签库 | 查看标签使用频率、合并同义标签、禁用滥用标签 |
| **样式/脚本审核** | 审核VIP作者提交的自定义CSS/JS | 通过 / 退回（附原因）/ 预览效果 |

#### 9.2.3 用户管理
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **用户列表** | 搜索/筛选所有用户（按注册时间、状态、角色） | 查看详情、重置密码、封禁/解封、授予/撤销角色 |
| **角色管理** | 管理用户角色（普通用户、作者、VIP作者、仲裁员、管理员） | 分配/移除角色，设定权限边界 |
| **实名认证审核** | 查看待审核的实名认证申请 | 通过 / 退回（附原因） |
| **作者认证** | 查看申请成为“认证作者”的用户 | 审核通过 / 退回 |

#### 9.2.4 财务与收益
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **平台收入总览** | 查看平台总收入（抽成）、月度趋势图、收入来源分布 | 导出报表（CSV/Excel） |
| **作者提现管理** | 查看所有作者的提现申请列表 | 审核通过 / 退回（附原因） / 标记已支付 |
| **打赏流水** | 查看全站打赏记录，按作品/作者/时间筛选 | 异常打赏标记 |
| **服务器成本公示** | 展示当月服务器账单与请求量，计算单次请求成本 | 编辑公示内容、发布财务透明报告 |
| **作者收益结算** | 查看每个作者的累计收益、待结算金额 | 手动触发结算（或由定时任务自动执行） |

#### 9.2.5 社区治理
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **举报处理** | 查看所有举报（按类型/严重程度/状态筛选） | 查看详情、判定成立/不成立、执行对应处罚 |
| **仲裁员管理** | 管理社区仲裁员名单（查看、任免） | 任命/撤销仲裁员、查看仲裁记录 |
| **社区公约管理** | 编辑社区公约（版本化，保留历史记录） | 提交修订草案 → 公示期 → 生效 |
| **公共差投票管理** | 发起/管理“价格上限”投票 | 设定投票时间、投票权重、公示结果 |

#### 9.2.6 系统监控
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **服务器状态** | 实时CPU/内存/磁盘/网络流量监控图表 | 无（只读） |
| **API性能** | 各接口P50/P95/P99延迟、QPS、错误率 | 无（只读） |
| **限流日志** | 查看触发限流的IP列表及详细请求日志 | 手动解封（如有误判） |
| **爬虫黑名单** | 管理被判定为商业爬虫的IP/网段 | 查看黑名单、手动添加/移除 |
| **操作日志** | 所有管理员操作的完整审计日志（含时间、操作人、内容） | 导出（仅站长） |

#### 9.2.7 系统设置
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **站点配置** | 站点名称、SEO标题/描述、Logo、Favicon | 在线编辑 |
| **注册设置** | 是否开放注册、邀请码机制、邮箱验证开关 | 开关配置 |
| **定价设置** | 平台抽成比例、最低提现金额、价格上限范围 | 修改（需确认） |
| **审核设置** | 是否开启先审后发、敏感词库、自动过滤规则 | 编辑敏感词列表、调整规则 |
| **存储设置** | 文件存储路径、允许的最大上传体积、图片尺寸限制 | 修改（需重启服务） |

## 10. 作者面板（Author Panel）

### 10.1 概述
作者面板是创作者的专属工作台，覆盖**从创作、发布、营销到收益管理**的全流程。

> 设计原则：一切操作应尽量减少摩擦，所有数据对作者完全透明。

### 10.2 面板结构

#### 10.2.1 仪表盘（Dashboard）
- **创作概览**：
  - 作品总数 / 总字数 / 总阅读量 / 总收益
  - 最近7天新增读者数 / 评论数 / 打赏数
  - 待处理事项（待审核作品、新评论、新粉丝等）
- **快捷入口**：
  - 新建作品 · 导入作品 · 查看收益 · 导出作品

#### 10.2.2 作品管理
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **作品列表** | 显示所有作品（封面、标题、分类、状态、阅读量、收益） | 点击进入详情、编辑、下架、删除 |
| **新建作品** | 填写标题、分类、简介、定价、封面图 | 保存草稿 / 提交审核 / 直接发布（视权限） |
| **章节管理** | 查看/编辑/删除章节列表，拖拽调整章节顺序 | 新增章节、批量导入章节、调整顺序 |
| **编辑器** | 富文本/Markdown双模式编辑，支持LaTeX公式、颜色标记 | 实时预览、保存、发布 |
| **批量操作** | 批量导入章节（上传ZIP/MD）、批量设置定价、批量修改状态 | 执行批量操作 |
| **版本历史** | 查看作品的修改历史（仅保留最近10个版本） | 回滚至指定版本 |

#### 10.2.3 作品样式定制（美化）
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **预设主题** | 选择平台提供的预设样式（亮色/暗色/护眼/极简等） | 一键应用 |
| **基础定制** | 调整背景色、文字颜色、标题颜色、字体、字号、行距 | 取色器选择 + 保存 |
| **高级定制（VIP）** | 自定义CSS输入框 + 自定义CSS预览 | 提交审核 → 审核通过后生效 |
| **动画定制（VIP）** | 选择平台预置动画（淡入、滑动、发光等）或提交自定义JS | 提交审核 → 审核通过后生效 |
| **封面/头图管理** | 上传/更换作品封面图、头图 | 上传 → 重新编码存储 → 自动裁剪缩略图 |

#### 10.2.4 收益管理
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **收益总览** | 查看累计收益、待结算金额、已提现金额 | 无 |
| **收入明细** | 查看每一笔收入（订阅、打赏）的来源与时间 | 筛选（按时间/作品）、导出明细 |
| **提现申请** | 发起提现（输入金额、选择提现方式） | 提交 → 待审核 → 审核通过后到账 |
| **提现历史** | 查看所有历史提现记录及状态 | 无 |

#### 10.2.5 读者互动
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **评论管理** | 查看所有作品的读者评论（按最新/热门/待回复排序） | 回复评论、点赞评论、举报评论 |
| **数据统计** | 作品阅读量、读者留存、读者画像（年龄/阅读偏好/来源渠道） | 查看图表、导出数据 |
| **粉丝管理** | 查看收藏/关注自己作品的读者列表 | 查看、发私信（如有此功能） |

#### 10.2.6 作品导入/导出
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **作品导出** | 选择作品 → 选择格式（ZIP/MD/EPUB/TXT） → 生成下载链接 | 一键导出 |
| **作品导入** | 上传ZIP/MD/TXT文件 → 系统自动解析 → 预览确认 → 导入 | 上传 → 预览 → 确认导入 |
| **导入历史** | 查看历史导入记录 | 无 |

#### 10.2.7 个人设置
| 子模块 | 功能 | 操作 |
| :--- | :--- | :--- |
| **个人资料** | 修改头像、昵称、简介、社交链接 | 在线编辑 |
| **实名认证** | 提交身份证信息进行实名认证（政治文学区访问前置条件） | 提交 → 待审核 |
| **VIP开通** | 查看VIP权益、付费开通VIP | 付费 → 开通 |
| **通知设置** | 设置评论通知、打赏通知、审核结果通知等 | 开关配置 |
| **绑定设置** | 绑定/解绑邮箱、手机号 | 验证后绑定/解绑 |

#### 10.2.8 授权与合规
- **作品授权声明**：作者可选择作品授权范围（仅平台发布/可转载需署名/全部保留等）。
- **版权声明页**：自动生成该作品的版权声明，置于作品页底部。
- **申诉通道**：若作品被误判下架，可提交申诉，交由仲裁员复核。

## 11. 站长面板与作者面板的交互关系

| 操作 | 触发方 | 处理方 | 结果 |
| :--- | :--- | :--- | :--- |
| 作者提交新作品 | 作者面板 | 站长面板·作品审核 | 通过 → 作品上线；退回 → 作者收到退回原因 |
| 作者提交VIP样式 | 作者面板 | 站长面板·样式审核 | 通过 → 样式生效；退回 → 作者收到退回原因 |
| 作者发起提现 | 作者面板 | 站长面板·提现管理 | 通过 → 进入支付流程；退回 → 作者收到退回原因 |
| 读者举报评论 | 读者（前端） | 站长面板·举报处理 | 判定成立 → 执行处罚；不成立 → 驳回 |
| 作者申诉下架作品 | 作者面板·申诉通道 | 站长面板 + 仲裁员 | 申诉成功 → 恢复上线；失败 → 维持原判 |
| 站长调整平台抽成比例 | 站长面板·系统设置 | 全平台生效 | 所有作者看到新的抽成比例 |

## 12. 面板设计交互原则

| 原则 | 说明 |
| :--- | :--- |
| **权限最小化** | 站长面板的每个子模块都需独立权限控制，不同管理员可分配不同权限 |
| **操作留痕** | 所有修改/审核/删除操作均记录操作人、时间、操作内容，支持审计 |
| **二次确认** | 涉及删除、封禁、资金操作的高危操作，需弹窗二次确认 |
| **透明告知** | 作者面板的所有操作结果（审核通过/退回/下架）均需附带明确原因说明 |
| **移动端适配** | 作者面板需响应式设计，确保在移动端也能正常使用（特别是审核功能） |
